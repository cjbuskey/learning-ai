The Trace That Saved the Inning

A Mets Fan's Guide to Langfuse

The Pitch

Built a two-step legal-brief summarizer (extract claims → write summary) instrumented end-to-end with Langfuse. Every Claude call is a generation span; every step is a named observation; every brief produces one trace with two scores attached.

PIPELINEextract → summarize → judge
MODELclaude-haiku-4-5
TRACES3 briefs × 1 trace each
SCOREScompression_ratio + faithfulness

The Lineup: Two Layers of Spans

The Langfuse skill says prefer framework integrations over manual spans. So:

1. AnthropicInstrumentor — auto-captured generations

One line — AnthropicInstrumentor().instrument() — patches the Anthropic SDK so every messages.create() becomes a generation span with model name, input, output, and token counts. Zero hand-written wrapper code.

2. @observe — named structural spans

Decorators on extract_claims, write_summary, judge_faithfulness, and process_brief. These name the business logic — Langfuse can't tell which step is which without me.

The split: instrumentor handles the what (LLM mechanics) — @observe handles the why (which step in your pipeline). Layered context with minimal code.

The Walk-Off: Observability Caught the Bug

First run: brief-001 and brief-002 both scored 0.0 faithfulness. Without traces, my next move would have been "the summarizer is hallucinating, fix the prompt."

Opened the Langfuse UI, clicked into the judge-faithfulness span. The raw model output was:

"0.75\n\nThe summary accurately captures most"
finish_reason: "length"

The judge had returned a correct score. max_tokens=10 truncated the trailing prose. float(text.strip()) raised ValueError on the partial sentence. The except branch returned 0.0.

The bug was in my parser, not the model. A print()-only world would have shown me three numbers — 0.0, 0.0, 0.92 — and pointed me at the wrong fix entirely. The trace was the single source of truth that decided which component to debug.

Fix: regex grabs the first number it finds, regardless of trailing prose.

m = re.search(r"\d*\.?\d+", text)
return float(m.group()) if m else 0.0

Interview Hook

"I built a two-step legal summarization pipeline with Langfuse traces and an LLM-judge faithfulness score. On the first run two of three traces scored 0.0 faithfulness. Looking at the raw judge output in Langfuse, the model had returned correct scores like '0.75' but max_tokens=10 truncated the trailing prose and my parser threw a ValueError on the partial sentence — falling through to a 0.0 default. The takeaway: observability isn't just for performance — it's the single source of truth that decides which component you debug."
